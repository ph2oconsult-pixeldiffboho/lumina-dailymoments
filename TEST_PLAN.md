# Real-World Testing Plan

This plan is designed to observe natural usage patterns and capture raw feedback from real families using the current version of the application.

## 1. Test Setup

*   **Number of Users:** 5–10 families (Parent + Child).
*   **Duration:** 7 consecutive days (one full journey cycle).
*   **Initial Focus:** The **"Feeling nervous"** journey. It is a common, high-frequency emotional state that provides a clear entry point for testing the 7-day rhythm.

## 2. What to Observe

We are looking for friction points and natural behavior, not feature requests.

*   **Retention:** Does the family return on Day 2? Do they make it to Day 4 (the "hump")?
*   **Hesitation:** On which screens do users pause for more than 10 seconds before acting? (e.g., Is the "Parent Guidance" confusing or helpful?)
*   **Disengagement:** At what point in the 7-day cycle or within a specific session do they close the app?
*   **Physical Interaction:** How is the phone held? Is it between them, or does the parent hold it away? (Observed via video or self-report).

## 3. Simple Feedback Questions

At the end of the 7 days, ask these three questions:

1.  **When did using this feel like effort?** (Looking for friction in the UI or the emotional "ask").
2.  **When did it feel easy?** (Looking for the moments where the app "disappeared").
3.  **When did you *not* want to use it?** (Looking for timing issues or mood mismatches).

## 4. Success Criteria

We will consider this version of the product successful if:

*   **Multi-Day Return:** At least 60% of test families return for 3 or more days without prompting.
*   **Natural Integration:** Feedback suggests the app felt like a "quiet moment" rather than a "digital chore" or a "lesson."
*   **Low Cognitive Load:** Users can navigate the steps without asking "What do I do here?"

---

**CRITICAL RULE:** No optimizations, redesigns, or new features will be implemented based on this plan until the 7-day observation period is complete.
